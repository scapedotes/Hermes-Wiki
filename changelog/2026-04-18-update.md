# Hermes Agent Changelog (2026-04-17 ~ 2026-04-18)

410 post-release commits and 56 features since v0.10.0. Key themes include **Architectural Refactoring** (Transport ABC), **Extensibility** (Shell Hooks, New Providers), and **Observability** (subagent tracing, account limits).

---

## Architectural Changes

### Provider Transport ABC (Major Architectural Refactoring)

Added `agent/transports/` directory (1217 lines, 7 modules), abstracting the API data path for all providers with a unified `ProviderTransport` ABC:

```
agent/transports/
├── base.py              # ProviderTransport ABC (89 lines)
├── types.py             # NormalizedResponse (142 lines)
├── __init__.py          # Registry + get_transport() lazy discovery (51 lines)
├── anthropic.py         # AnthropicTransport (177 lines)
├── chat_completions.py  # ChatCompletionsTransport (387 lines)
├── codex.py             # ResponsesApiTransport (217 lines)
└── bedrock.py           # BedrockTransport (154 lines)
```

**Core Abstraction**: `convert_messages → convert_tools → build_kwargs → normalize_response`, plus three optional hooks (`validate_response`, `extract_cache_stats`, `map_finish_reason`).

This replaced 10+ scattered `if api_mode == "anthropic_messages": ... elif ...` conditional branches in `run_agent.py`. Auxiliary clients also migrated to Transport (compression, memory flush, session summary paths).

For details, see [[provider-transport-architecture]].

### Shell Hooks

Hook callbacks are no longer limited to Python. Users can declare shell scripts in `config.yaml`:

```yaml
hooks:
  pre_tool_call:
    - command: /path/to/my-hook.sh
  subagent_stop:
    - command: /path/to/audit.sh
```

Scripts receive JSON events from stdin and return JSON decisions to stdout. Key design features: `subprocess.run(shell=False) + shlex.split` to prevent shell injection; user consent requested and stored in an allowlist for the first use of each `(event, command)` pair; `hermes hooks list/test/revoke/doctor` CLI subcommands; Claude Code-compatible response format. A new `subagent_stop` event has been added.

### Delegate Orchestrator + Cross-Agent File Coordination

`delegate_task` now includes `role='leaf'|'orchestrator'`, enabling multi-level delegation when combined with `delegation.max_spawn_depth` (default 1 for flat, 2-3 for nested).

Concurrent subagents now share a file state view. The parent agent can see all file operations performed by its subagents during summarization, preventing loss of concurrent patches.

## Observability

### Subagent Spawn Observability Overlay (TUI)

A **real-time + post-mortem audit** view for recursive `delegate_task` fan-outs. The Ink overlay supports nested subagent trees of arbitrary depth and width. Each subagent event now includes:
- `subagent_id`, `parent_id`, `depth`, `model`, `tool_count`
- `subagent.complete` additionally carries input/output/reasoning tokens, cost, API calls, files_read/files_written

### /usage Account Limits

The `/usage` command now appends **provider-side account quota/limit** information below the token table. The CLI uses `concurrent.futures.ThreadPoolExecutor(max_workers=1)` with a 10s timeout, while the gateway uses `asyncio.to_thread`, ensuring slow providers do not impact interactivity. The new `agent/account_usage.py` module (326 lines, `fetch_account_usage` + `render_account_usage_lines`) provides per-provider abstraction.

### Dashboard Enhancements

- Track actual API call count per session
- Added "Update Hermes" and "Restart Gateway" buttons
- Theme system + plugin system further refined

## Platform Enhancements

### Cross-Platform Native Plugin Slash Commands

Plugin commands registered with `register_command()` are now natively displayed on each gateway platform: Discord native slash selector / Telegram BotCommand menu / Slack `/hermes` subcommands. A new `args_hint` parameter supports argument selectors.

**Decision-making command hook**: The `command:<name>` hook can return `{"decision": "deny|handled|rewrite|allow"}` to intercept before core processing. This is backward compatible with fire-and-forget telemetry hooks.

### WeCom

QR code scanning authentication + interactive setup wizard (`hermes_cli/gateway.py:_setup_wecom` → `gateway.platforms.wecom.qr_scan_for_bot_info`), eliminating the need for manual bot credential configuration.

### Feishu / Mattermost / Matrix

- **PLATFORM_HINTS**: System prompt overrides for Matrix, Mattermost, Feishu
- **Feishu @mention Retention**: Inbound messages now retain @mention context
- **Feishu Streaming Newline Fix**: Streaming edits no longer prepend extra blank lines

### Slack / Gateway General

- `SLACK_REACTIONS` environment variable to toggle reaction lifecycle
- MEDIA: Tag support for automatic extraction of PDF, document, archive file extensions
- `require_mention_channels` configuration
- Global `security.allow_private_urls` toggle — addresses scenarios like OpenWrt/TUN proxies/enterprise VPNs/Tailscale (DNS resolves to private IPs in 198.18.0.0/15, 100.64.0.0/10). Cloud metadata endpoints (e.g., 169.254.169.254) are always blocked.

## Session Storage

### Automatic Prune + VACUUM on Startup

`state.db` previously grew indefinitely. Heavy users reported 384MB/982 sessions impacting performance. As of v2026.4.18+, the following are automatically performed on startup:
- Added `state_meta` table to record last run timestamp
- Cross-process idempotent (executes only once under the same `HERMES_HOME`)
- Default once daily (`min_interval_hours=24`), cleaning up sessions older than 90 days (`retention_days=90`)
- Smart VACUUM: Only executes if `pruned > 0`; empty cleanups do not waste I/O
- Never throws exceptions
