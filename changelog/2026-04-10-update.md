# Hermes Agent Changelog (2026-04-09 ~ 2026-04-11)

293 commits, 35+ new features. Key highlights are below.

---

## Architectural Changes

### Context Engine Pluginization

**Core Change**: Context management has been refactored from a hardcoded `ContextCompressor` into a pluggable `ContextEngine` ABC (`agent/context_engine.py`).

```yaml
# config.yaml — Switch context engine
context:
  engine: "compressor"   # Default; set to plugin name to switch (e.g., "lcm")
```

The ABC requires implementing 3 core methods:
- `name` — Engine identifier
- `should_compress(prompt_tokens)` — Whether compression is needed
- `compress(messages, current_tokens)` — Executes compression, returns new message list

Optional methods: `on_session_start/end`, `get_tool_schemas` (engines can expose tools to the agent), `update_model` (updates when model switches).

The built-in `ContextCompressor` now inherits from `ContextEngine` and is the default implementation. Third-party plugins are located in the `plugins/context_engine/<name>/` directory.

### Unified File Synchronization (file_sync.py)

New module `tools/environments/file_sync.py`: Provides unified file synchronization for SSH/Modal/Daytona backends, based on mtime+size change tracking, supporting deletion detection and transaction rollback. Docker/Singularity do not require this as they use bind mounts.

### Background Process Monitoring (watch_patterns)

The `terminal` tool now includes a `watch_patterns` parameter. Background process output matching specified strings will notify the agent in real-time:

```python
terminal(command="pytest -v", background=True, watch_patterns=["ERROR", "FAIL", "listening on port"])
```

Built-in rate limiting (max 8 notifications in a 10-second window). Continuous overload for 45 seconds automatically disables the feature to prevent notification storms.

### Discord

| Feature | Description |

| **require_mention** | Requires `@` mention to respond in channels (default true) |
