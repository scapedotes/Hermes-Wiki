## New Features

### Structured API Error Classification + Intelligent Failover

**commit**: `1a3ae6ac` (#6514) | **New file**: `agent/error_classifier.py` (789 lines)

Replaced dispersed string matching logic in `run_agent.py` with a unified error classifier. All API errors are now categorized into 13 `FailoverReason` types, each corresponding to a different recovery strategy:

| Error Type | Recovery Strategy |
|---|---|
| `auth` | Refresh/Rotate credentials |
| `billing` | Immediately switch Provider |
| `rate_limit` | Rotate after backoff and wait |
| `context_overflow` | Compress context |
| `timeout` | Rebuild client + retry |
| `model_not_found` | Fallback to other models |
| `server_error` | Retry |

Classification results include structured recovery hints (`retryable`, `should_compress`, `should_rotate_credential`, `should_fallback`). The retry loop directly reads these fields, eliminating redundant error parsing.

### Unified Execution Layer (spawn-per-call)

**commit**: `d684d7ee`

Terminal environment architecture refactor. All 6 backends (local, docker, ssh, singularity, modal, daytona) are unified into a "spawn `bash -c` process per command" model:

- Capture a session snapshot once during initialization (env vars, functions, aliases)
- Re-source the snapshot before each command
- CWD is persisted across calls via stdout markers (remote) or temporary files (local)

This replaces the previous persistent shell solutions individually implemented by each backend (`persistent_shell.py` has been deleted).

### Tool Result Persistence (3-Layer Protection)

**New files**: `tools/tool_result_storage.py`, `tools/budget_config.py`

Three mechanisms to prevent large tool outputs from overflowing the context window:

| Layer | Location | Mechanism |
|---|---|---|
| 1. In-tool truncation | Within each tool | Tool self-truncates output |
| 2. Single result persistence | `tool_result_storage.py` | Exceeds threshold (default 100K chars) → writes to `/tmp/hermes-results/`, context only retains 1500 char preview + file path |
| 3. Turn-aggregated budget | `budget_config.py` | Total tool results for a single turn exceed 200K chars → largest non-persisted result overflows to disk |

`read_file` is pinned to a threshold of `∞` to prevent a "persist → read → re-persist" infinite loop.

## Important Fixes

### Stability

| Fix | Commit | Impact |
|---|---|---|
| Session deletion changed to orphan | `a9409990` (#6513) | When deleting/cleaning sessions, child sessions (generated by compression splits) are no longer cascade-deleted but orphaned (`parent_session_id` set to NULL), preserving searchable history. |
| Empty response retries 3 times | `1eabbe90` (#6488) | Automatically retries when model returns empty content, no longer fails directly. |
| Credential exhaustion TTL reduced to 1 hour | `b408379e` (#6504) | Reduced from 24 hours to 1 hour, allowing quicker re-attempt after a key is marked invalid. |
| Compression tail protection min_tail=3 | `d12f8db0` | Forces retention of at least 3 tail messages during compression, based on token budget instead of a fixed message count. |
| Compression quality improvement | `85670314` | Named constants, tool call tracking, degradation alerts, preserving more critical information after compression. |

## Architectural Changes Summary

1.  **Error Handling**: From 190 lines of scattered string matching in `run_agent.py` → centralized to a structured classifier in `error_classifier.py`
2.  **Terminal Execution**: From each backend individually maintaining a persistent shell → unified spawn-per-call + session snapshot
3.  **Tool Results**: Added 3 layers of overflow protection; large outputs no longer overwhelm the context window
4.  **Session Management**: Deletion operations changed from cascade delete to orphan, protecting historical data integrity
