## Architectural Changes

### Platform Adapter Pluginization (Major Architectural Change)

`gateway/platform_registry.py` introduces `PlatformRegistry` singleton + `PlatformEntry` dataclass, allowing anyone to integrate new platforms (e.g., IRC, Viber, Line) as **pure plugins**, without modifying the gateway's core code.

**12 integration points fully covered**: webhook delivery, PLATFORM_HINTS, `get_connected_platforms`, cron delivery, dynamic toolset generation, setup wizard, unified settings flow, etc. Bundled plugin platforms (e.g., IRC) are automatically loaded at startup.

New PluginContext API:

```python
def register(ctx):
    ctx.register_platform(
        name="irc",
        label="IRC",
        adapter_factory=create_irc_adapter,
        check_fn=check_irc_available,
        validate_config=validate_irc_config,
        required_env=["IRC_NICK", "IRC_PASS"],
        install_hint="pip install ...",
    )
```

**IRC Reference Implementation** (`plugins/platforms/irc/`): Pure stdlib asyncio, zero external dependencies, TLS, PING/PONG, nick collision, NickServ auto-authentication, channel addressing (`nick: msg`), Markdown stripping, message fragmentation, interactive setup wizard.

See [[messaging-gateway-architecture]].

## New Module: Curator — Background Skill Maintenance

`agent/curator.py` (869 lines) + `hermes_cli/curator.py` (235 lines) + `tools/skill_usage.py` — **Auxiliary model-driven** background maintenance mechanism, periodically reviewing agent-created skills, tracking usage, and archiving idle skills based on state machine transitions.

**Invariants**:
- Never touches bundled / hub-installed skills (double filtered)
- Never auto-deletes — only archives; can be restored with `hermes curator restore <skill>`
- Pinned skills skip all automatic transitions
- Uses aux client, never contaminates the main session's prompt cache

**Trigger**: Inactivity-triggered (no cron daemon). Checks at CLI/Gateway startup, runs only if `interval_hours` (default 168h = 7 days) + `min_idle_hours` (default 2h) are met. Gateway also hooks into the cron-ticker thread for periodic checks.

**State Machine**: `active → stale → archived`, reverts to active upon reuse. Pure functional, no LLM calls. AIAgent only intervenes via fork when integration of overlaps + drift patching is required.

**CLI**: `hermes curator {status,run,pause,resume,pin,unpin,restore}`, corresponding to the `/curator` slash command.

See [[skills-system-architecture]].

## Memory System

### `on_session_switch` Provider Hook

Previously, providers only received `session_id` once during initialization. However, `session_id` can be **reassigned** in scenarios such as `/resume`, `/branch`, `/reset`, `/new`, and context compression — providers were unaware, leading to subsequent writes going into incorrect session records.

`agent/memory_manager.py:on_session_switch()` now calls all providers' `on_session_switch(new_session_id, parent_session_id, reset, **kwargs)` when `session_id` changes, allowing providers to refresh their cached per-session states.

See [[memory-system-architecture]].
