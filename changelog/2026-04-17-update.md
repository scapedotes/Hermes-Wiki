## Nous Tool Gateway

Routes API calls for web search, TTS, browser, and image generation to a unified gateway hosted by Nous, eliminating the need for users to provision individual API keys. Switched from hidden environment variables to subscription-based auto-detection:

```yaml
# config.yaml — Opt-in by tool category
web:
  use_gateway: true
tts:
  use_gateway: true
```

After logging in with `hermes model`, available tools are displayed, and users can choose to enable all / only unconfigured / skip.

## Context Compression v3

### Three-Phase Preprocessing (No LLM Calls, Zero Token Cost)

| Pass | Operation | Effect |
|------|-----------|--------|
| Pass 1 | MD5 Deduplication | If a file is read 5 times, only the latest copy is kept, older copies are marked `[Duplicate tool output]`. |
| Pass 2 | Smart Collapse | Replaces old tool outputs with informative single-line summaries (e.g., `[terminal] ran npm test → exit 0, 47 lines`). |
| Pass 3 | tool_call Argument Truncation | Truncates arguments >500 characters to 200 (fixes `write_file` 50KB argument residue). |

### Summary Template Upgrade (v2 → v3)

- `Progress > Done` (free text) → `Completed Actions` (forced numbering `N. ACTION target — outcome [tool: name]`).
- Added `Active State` section (working directory/branch/test status/running processes).
- Removed `Tools & Patterns` section (information integrated into Completed Actions as `[tool: name]`).
- Summary budget reduced from `budget × 2` to `budget × 1.3`.

### Debouncing + Tail Protection

- **Anti-Thrashing**: Skips if two consecutive compressions save <10%.
- **Failure cooldown**: RuntimeError 600s / Transient 60s (distinguishes configuration errors vs. temporary failures).
- **v0.10.0 Fix**: Added `_ensure_last_user_message_in_tail()`, ensuring the last user message is never pushed into the summary area (#10896).

## Plugin System Enhancements

### Plugin Namespaced Skills

Plugins can register namespaced skills (`plugin:skill`) to avoid conflicts with built-in skills:

```python
ctx.register_skill(name="deploy", path=..., description="...")
# Actually saved as "myops:deploy", agent calls skill_view("myops:deploy")
```

- Not indexed in system prompts (zero token overhead).
- Bundle context banner lists sibling skills from the same plugin.
- Comprehensive protection: disable checks, platform compatibility, injection mode scanning.

### pre_tool_call Blocks Tool Execution

`pre_tool_call` hook upgraded from notification-based to blocking:

```python
def my_hook(tool_name, args):
    if "rm -rf" in args.get("command", ""):
        return {"action": "block", "message": "Blocked by policy"}
```

The first `{"action": "block"}` takes effect, the tool execution is skipped, and the message is returned to the model as a tool result.

### register_command() (v0.10.0)

Plugins can register custom slash commands:

```python
ctx.register_command(name="deploy", description="...", handler=my_handler)
```

Automatically appears in Telegram bot menus and CLI autocompletion. Conflict detection with built-in commands.

### dispatch_tool() (v0.10.0)

Plugin slash command handlers can dispatch tool calls via the registry, automatically injecting the parent agent context:

```python
result = ctx.dispatch_tool("delegate_task", {"task": "..."})
```

### Dashboard Plugin System + Theme System (v0.10.0)

Plugins can add custom tabs to the Web Dashboard (`plugins/<name>/dashboard/manifest.json` + `dist/index.js`). A Dashboard theme system has also been added, supporting real-time switching.

## Tool Auto-Discovery

`tools/registry.py` adds `discover_builtin_tools()`: AST scans `tools/*.py`, automatically importing any file with a top-level `registry.register()` call. Adding new tools no longer requires manual modification of `model_tools.py`'s import list.

## Gateway Proxy Mode

Gateway and Agent can be deployed separately. Typical scenario: Gateway runs in Docker handling Matrix E2EE encryption, while the Agent runs on a macOS host accessing local files and memory.

```yaml
gateway:
  proxy_url: "http://host.docker.internal:8080"
```

Forwarded via HTTP + SSE streaming, with the `X-Hermes-Session-Id` header ensuring session continuity across requests.
